Tại Sao Chọn EntityData + ModuleData?

Design Decision
❌ Cách tôi thử trước
MonoBehaviour giữ data trực tiếp
→ data và logic dính chặt, không tách được
Mỗi loại item = 1 Prefab riêng
→ 100 item = 100 Prefab, thay đổi data phải mở Unity
if/switch để check loại item
→ thêm loại mới = sửa code ở nhiều chỗ
Duplicate code giữa các item tương tự
→ bug fix 1 chỗ, quên 5 chỗ kia
Scale đến ~30 item → khó maintain
→
✓ Giải pháp chọn
ScriptableObject = pure data asset
→ chỉnh data trong Inspector, không cần code
ModuleData[] = behavior opt-in
→ ghép module nào = có behavior đó, không ghép = không có
Thêm loại item mới = tạo asset mới
→ không cần viết code, không break existing
EntityData shared across instances
→ 1000 instance cùng loại dùng chung 1 data asset
200+ item · không cần refactor lại
