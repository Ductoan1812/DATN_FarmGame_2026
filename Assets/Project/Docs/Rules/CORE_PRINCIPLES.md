# Core Principles

Priority:
1. Correctness
2. Stability
3. Readability
4. Maintainability
5. Performance

Project Goals:
- maintainable
- modular
- readable
- predictable
- fast iteration
- low technical debt
- solo dev friendly

Avoid:
- overengineering
- unnecessary abstraction
- premature optimization
- giant refactors
- hidden side effects

Prefer:
- simple maintainable solutions
- incremental changes
- isolated fixes
- compatibility-safe updates
