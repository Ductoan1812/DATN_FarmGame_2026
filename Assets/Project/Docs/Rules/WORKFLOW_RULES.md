# Workflow Rules

Before coding:
1. Analyze architecture
2. Find root cause
3. Reuse existing systems
4. Minimize disruption
5. Preserve compatibility

Debugging:
1. Read logs first
2. Verify assumptions
3. Explain issue cause
4. Apply safest fix
5. Mention regression risks

Avoid:
- random trial/error
- rewriting working systems
- unrelated changes
